# 校园跑腿后台管理系统

## 功能介绍

这是一个完整的后台管理系统,用于监控和管理校园跑腿平台的数据。

### 主要功能

1. **数据统计**
   - 总用户数
   - 总订单数
   - 已完成订单数
   - 待接单数

2. **用户管理**
   - 查看所有用户列表
   - 搜索用户(按用户名、昵称)
   - 分页浏览
   - 查看用户详细信息(包含订单统计)

3. **订单管理**
   - 查看所有订单列表
   - 按状态筛选订单
   - 搜索订单(按标题、描述)
   - 分页浏览
   - 查看订单详细信息

## 快速开始

### 1. 安装依赖

```bash
cd admin-system
npm install
```

### 2. 配置环境变量

编辑 `.env` 文件,配置数据库连接信息:

```env
PORT=3001
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的MySQL密码
DB_NAME=errand_platform
```

### 3. 启动服务

```bash
npm start
```

或使用开发模式(支持热重载):

```bash
npm run dev
```

### 4. 访问系统

在浏览器中打开: `http://localhost:3001`

## 技术栈

- **后端**: Node.js + Express
- **数据库**: MySQL
- **前端**: 原生 HTML/CSS/JavaScript
- **接口**: RESTful API

## API 接口

### 统计数据
```
GET /api/stats
```

### 用户管理
```
GET /api/users              # 获取用户列表
GET /api/users/:id          # 获取用户详情
```

### 订单管理
```
GET /api/orders             # 获取订单列表
GET /api/orders/:id         # 获取订单详情
```

### 查询参数

- `page`: 页码(默认1)
- `pageSize`: 每页数量(默认10)
- `search`: 搜索关键词
- `status`: 订单状态(pending/accepted/completed/cancelled)

## 系统截图

### 统计面板
- 实时显示平台核心数据
- 四个统计卡片展示关键指标

### 用户管理
- 表格展示所有用户信息
- 支持搜索和分页
- 点击查看用户详情和订单统计

### 订单管理
- 表格展示所有订单信息
- 支持状态筛选
- 支持搜索和分页
- 点击查看订单完整信息

## 注意事项

1. 确保数据库 `errand_platform` 已创建并初始化
2. 后台管理系统端口默认为 3001,避免与主服务(3000)冲突
3. 如需修改端口,请编辑 `.env` 文件中的 `PORT` 配置
4. 系统直接连接生产数据库,请谨慎操作
